# Wattly — 電力SaaS（構想中）

## プロジェクト概要

電力業界向けSaaS。まずはアフィリエイトメディアとしてスタートし、ドメイン知識とドメインパワーを蓄積してからSaaS事業の方向性を決定する。

### 戦略
1. **Phase 0**: 電力アフィリエイトメディア運営でドメイン知識 + ドメインパワー + 収益を獲得
2. **Phase 1**: 蓄積した知識をもとにSaaSの方向性を決定（月3〜5万PV到達が目安）
3. **Phase 2**: MVP開発（メディアの読者が見込み客になる）

**Casinohubとの違い**: Casinohubは「作るべきSaaSが明確（チップ管理）」なので即開発。Wattlyは「何のSaaSを作るかが未定」なのでメディア運営で知見を貯めてから判断する。ただしメディア運営中に明確なペインが見つかったらPVに関係なく即動く。

## プロジェクト構成

```
wattly/
├── hp/               ← メディアサイト（Next.js）→ アフィリエイト + SEO
├── marketing/        ← 記事原稿
├── 事業運営/          ← 事業運営タスク管理
└── images/           ← ロゴ・素材
```

### URL設計
- 全ての記事を `/column/` 配下に統一する
- 例: `wattly.jp/column/electricity-bill-structure`
- 消費者向け・事業者向けの区別はURLではなくタグやカテゴリで管理

## 共通ルール

- **言語**: 日本語で作業・出力
- **金額**: 全て税込で記載し、表示時は「（税込）」を必ず付記する
- **日付**: YYYY-MM-DD 形式
- **コミットメッセージ**: 日本語、簡潔に
- **秘密情報**: `.env`, credentials, API keyは絶対にコミットしない

### ワークフロー
- **確認不要**: wattly配下のプロジェクトではコマンド実行・push 含め一切確認を求めず自律的に進める
- **「リリースして」= push まで含む**: コミット → push まで一気にやる

### 文体・トーン（記事・メディア）
- 同じ目線で語るトーンを使う
- 上から目線・偉そうな口調はNG
- 断言できないことは断言しない
- 競合の名指し・比較・批判はしない
- 電気に詳しくない一般消費者にもわかりやすく

### アフィリエイト導線設計
- ASP未登録のため今はリンクを貼らないが、**将来差し替えやすい構造**にしておく
- 具体的なサービス名を出すか「複数社を比較検討するのがおすすめ」の一文を自然に入れる
- CTA（比較表・シミュレーション等）を記事内に設置する想定で構成を組む

### セルフレビュー工程（必須）
全ての成果物を提出する前に、「ユーザーがさらに優秀になったバージョン」を模倣して一度レビュー・ブラッシュアップを行ってから提出すること。

### Ralph Loop開発フロー（PRD駆動の自律開発）

1. **PRD作成**: `PRD.md` にタスク一覧を作成
2. **PRD深掘り**: 各タスクを実装・テスト・検証・エッジケースまで含めたサブタスクに分解
3. **Ralph Loop実行**: PRDのタスクを上から順に消化。全タスクが `[x]` + 検証PASSまで止まらない
4. **完了タスクは即座に `[x]` 更新** → コミット → 次へ

### 進捗レポート出力（必須）

長時間作業中は、ユーザーが非同期で確認できるように以下のフォーマットで出力する:

**タスク完了時:**
```
[完了] Phase X > X.X タスク名
- 実装内容: 概要
- 検証結果: テスト結果 or ビルド結果
- 確認URL: （デプロイがある場合）
- 次のタスク: X.X タスク名
```

**ブロッカー発生時:**
```
[ブロック] Phase X > X.X タスク名
- 原因: 詳細
- 必要なアクション: ユーザーが何をすればいいか
- スキップして次のタスクに進みます
```

**Phase完了時:**
```
[Phase完了] Phase X: フェーズ名
- 完了: X/X タスク
- ブロック: Xタスク（内容）
- 確認URL: （あれば）
- 次のアクション: Phase X に進む or ブロッカー解消？
```

### SEO戦略（2層構造）

エネチェンジ・価格.comが「比較・おすすめ」系を押さえているため、正面衝突は避ける。**視座系（看板記事）と実用系の2層構造**で攻める。

**視座系（看板記事）: 月1〜2本、しっかり書く**
- 「問いを立てる」「前提を疑う」「構造を俯瞰する」切り口
- 例: 「電気代は本当に高いのか？」「電力自由化は誰のためだったのか」「なぜ電気料金はわかりにくいのか」
- 競合が書かない領域。AIでも量産できない（視座が必要）
- SNSシェア・被リンクでドメインパワーを育てる（検索流入ではなくSNS起点）
- 実用記事への内部リンクのハブになる → 実用記事のSEO評価を底上げ
- SaaS展開時の思想的土台にもなる（問題提起→だからツールを作った、の流れ）

**実用系: 効率的に数を揃える**
- 節約・比較・制度解説などの実用記事
- 視座系で育てたドメインパワー+内部リンクの恩恵でSEO上位を狙う
- アフィリエイト成約はこちらで担う
- ロングテール（個別電力会社レビュー等）と法人特化（高単価×低競争）を優先

**収益の流れ:**
```
視座系記事（SNS・被リンク） → ドメインパワー向上
  → 実用記事のSEO順位UP → アフィリエイト成約
視座系記事 → 内部リンク → 実用記事への回遊 → 成約
```

**独自シミュレーター（将来）** — 真似しにくい堀を築く
- 電気料金比較ツールをWeb上に構築（使用量・アンペア・地域→各社比較）
- 「ツール目的の流入」が生まれる。被リンクも集まりやすい
- これがそのままSaaS展開の種になる（法人向け高機能版等）

- 季節性: 1〜3月が最大需要期（引越しシーズン）
- 詳細: `marketing/コンテンツ計画.md`

### コンテンツ計画
- 視座系（看板記事）: 月1〜2本
- 実用系: 効率重視で数を揃える
- 目安: 視座系10本 + 実用系30本 = 計40本で月3万PV目標

### 収益シミュレーション（現実的な見通し）

| PV目標 | 必要記事数 | 期間 | 難易度 | 月間収益目安 |
|--------|----------|------|-------|-----------|
| 月3万PV | 30〜50記事 | 6〜9ヶ月 | やれる | 10〜15万円 |
| 月10万PV | 80〜120記事 | 12〜18ヶ月 | 努力次第 | 20〜30万円 |
| 月30万PV | 200記事+ | 18〜24ヶ月 | 相当厳しい | 60〜80万円 |

- エネチェンジ・価格.comがビッグKWを押さえているため30万PVは非常に難しい
- ただし電力は1件の報酬が高いので**PVの割に収益が出やすい**
- 太陽光・蓄電池案件なら1件2〜3万円。月5件成約で月10万超
- 法人向け比率を上げれば少ないPVでも高収益が狙える
